Content reports

>>>

GET /katello/api/content_reports/system_status
Show the latest subscription status for a list of content hosts that have reported their subscription status during a specified time period. Running this report with minimal parameters will return all status records for all reported content hosts.

Параметры

Имя параметра Описание
system_id
необязательно

Filters the results by the given content host UUID.

Validations:

  • string from 2 to 128 characters containing only alphanumeric characters, space, ‘‘, ‘-’ with no leading or trailing space..
organization_id
обязательно

ID организации

Validations:

  • string from 2 to 128 characters containing only alphanumeric characters, space, ‘‘, ‘-’ with no leading or trailing space..
status
необязательно

Filter results on content host status.

Validations:

  • Must be one of: valid, invalid, partial.
on_date
необязательно

Date to filter on. If not given, defaults to NOW. Results will be limited to status records that were last reported before or on the given date. Must be a date in the form of “YYYY-MM-DD” or “YYYY-MM-DD HH:MM:SS”.

Validations:

  • Must be a Date in the form of YYYY-MM-DD.
management_enabled
необязательно

Include only content-hosts with the management enabled attribute true or false

Validations:

  • boolean

GET /katello/api/content_reports/system_trend
Show a listing of all subscription status snapshots from content hosts which have reported their subscription status in the specified time period.

Параметры

Имя параметра Описание
system_id
обязательно

Filters the results by the given content host UUID.

Validations:

  • string from 2 to 128 characters containing only alphanumeric characters, space, ‘‘, ‘-’ with no leading or trailing space..
organization_id
обязательно

ID организации

Validations:

  • string from 2 to 128 characters containing only alphanumeric characters, space, ‘‘, ‘-’ with no leading or trailing space..
start_date
необязательно

Start date. Used in conjunction with end_date. Must be a date in the form of “YYYY-MM-DD” or “YYYY-MM-DD HH:MM:SS”.

Validations:

  • Must be a Date in the form of YYYY-MM-DD.
end_date
необязательно

End date. Used in conjunction with start_date. Must be a date in the form of “YYYY-MM-DD” or “YYYY-MM-DD HH:MM:SS”.

Validations:

  • Must be a Date in the form of YYYY-MM-DD.
hours
необязательно

Show a trend between HOURS and now. Used independently of start_date/end_date.

Validations:

  • Integer
